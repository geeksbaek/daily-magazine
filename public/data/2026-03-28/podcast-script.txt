안녕하세요, GEEK DAILY 매거진입니다. 2026년 3월 28일 토요일, 오늘의 테크 뉴스를 전해드립니다.

오늘의 커버 스토리는 AI 코딩 도구 전쟁의 새로운 국면입니다. Anthropic의 Claude Code가 2.1.86 버전으로 대규모 업데이트를 단행했고, 같은 날 OpenAI의 Codex도 플러그인 기능을 정식 출시했습니다. 두 진영이 동시에 대형 업데이트를 내놓으면서, AI 코딩 에이전트 시장의 경쟁이 본격적으로 달아오르고 있습니다.

먼저 Claude Code 2.1.86 업데이트부터 살펴보겠습니다. 이번 버전에는 무려 26가지 CLI 변경사항과 5가지 시스템 프롬프트 업데이트가 포함되었습니다. 가장 주목할 만한 변화는 세 가지입니다. 첫째, 프록시 세션 그룹핑을 위한 X-Claude-Code-Session-Id 헤더가 추가되었습니다. 기업 환경에서 여러 세션을 추적하고 관리하는 데 핵심적인 기능이죠. 둘째, debug-before-escalate 규칙이 도입되었습니다. 이건 에스컬레이션 전에 먼저 온콜 진단과 수정을 시도하도록 요구하는 규칙인데요, AI 에이전트가 문제를 만나면 바로 사람에게 넘기는 게 아니라, 스스로 먼저 디버깅을 시도하게 만든다는 점에서 에이전트의 자율성이 한 단계 올라간 셈입니다. 셋째, PowerShell 네이티브 지원이 추가되었습니다. 그동안 Windows 개발자들이 가장 많이 요청했던 기능 중 하나였는데, 이제 Windows 환경에서도 별도 설정 없이 바로 사용할 수 있게 되었습니다. 또한 hooks에 if 필드가 추가되어, 특정 bash 커맨드에만 선택적으로 hook을 적용할 수 있게 되었습니다. 권한 규칙 문법을 사용하기 때문에 훨씬 세밀한 제어가 가능해졌죠.

이제 OpenAI Codex 플러그인 이야기로 넘어가겠습니다. OpenAI가 코딩 에이전트 Codex에 플러그인 기능을 정식 출시했는데요, 이게 상당히 의미 있는 변화입니다. Slack, Figma, Notion, Gmail, Google Drive, Linear, GitHub 등 개발자들이 일상적으로 사용하는 거의 모든 협업 도구와 즉시 연동이 가능합니다. 플러그인은 크게 세 가지로 구성됩니다. Skills는 에이전트에게 워크플로를 설명하는 프롬프트이고, Apps는 외부 도구 연동, 그리고 MCP 서버는 원격 도구나 공유 컨텍스트를 제공합니다. 이 세 가지를 하나의 설치 가능한 번들로 묶어서 팀이나 프로젝트 간에 재사용할 수 있습니다. 특히 주목할 점은, Codex가 단순한 코딩 도구에서 개발 전체 스택을 아우르는 에이전트로 진화하고 있다는 겁니다. 코드 작성 전후의 기획, 리서치, 조율 업무까지 지원 범위가 확대되었거든요. Greg Brockman은 트위터에서 "Codex의 사용 사례는 Skills와 같다. 단, 인간을 위한 것이다"라고 언급하기도 했습니다. 한편, 중국의 Z.AI도 GLM-5.1 모델을 탑재한 Coding Plan을 출시하며 이 시장에 뛰어들었습니다. Claude Code나 OpenClaw에서 설정 파일만 수정하면 바로 전환할 수 있다고 하는데요, 204,800 토큰 컨텍스트 윈도우에 131,072 토큰 최대 출력을 지원합니다. AI 코딩 에이전트 시장이 삼파전 양상으로 확대되고 있는 셈이죠.

다음 뉴스입니다. 230년 역사의 철도 차량 제조 기업 STADLER가 OpenAI를 도입해 인상적인 성과를 내고 있습니다. 전 세계 650명 이상의 직원이 일하는 이 회사는 2023년부터 모든 컴퓨터 사용 직원이 AI를 쓰도록 했는데요, 결과가 놀랍습니다. 일반적인 지식 업무에서 30에서 40퍼센트의 시간 절감, 초안 작성 속도 평균 2.5배 향상, 소셜 미디어 같은 대량 업무에서는 최대 6배까지 빨라졌습니다. 일일 활성 사용률이 85퍼센트를 넘을 정도로 직원들의 자발적 채택률도 높고요. 125개 이상의 커스텀 GPT를 만들어 번역, 이메일, 문서 작성, 기술 지원 등에 활용 중입니다. Co-CEO Julia Stadler는 "반나절 걸리던 초안 작성이 20분으로 줄었다"고 말했습니다. 이제 STADLER는 다음 단계로 AI 에이전트를 핵심 워크플로에 통합하는 것을 추진하고 있습니다. 230년 된 제조 기업도 이 정도인데, AI 도입을 망설이는 기업이라면 한번 돌아볼 필요가 있겠죠.

VS Code 1.114가 릴리스되었습니다. GitHub Copilot 통합이 더욱 강화되었고, 새로운 편집기 기능과 TypeScript 지원 개선, 성능 최적화가 주요 변경사항입니다. 전 세계 수천만 개발자가 매일 사용하는 에디터인 만큼, 관심 있으신 분들은 업데이트 노트를 확인해보시기 바랍니다.

빅테크 소식도 있습니다. Windows 11 설치 시 Microsoft 계정 로그인을 강제하는 정책이 드디어 바뀔 수도 있습니다. Microsoft 부사장 Scott Hanselman이 직접 "나도 싫어한다, 해결 중이다"라고 언급했거든요. 기술적으로는 쉽게 제거할 수 있지만, 여러 부서가 계정 연동으로 얻는 데이터와 수익 때문에 내부 조율이 필요한 상황이라고 합니다. 아직 구체적인 일정은 없지만, 내부에서 논의가 진행 중이라는 것 자체가 의미 있는 신호입니다.

트위터 펄스, 이번 주 주목할 트윗들을 빠르게 살펴보겠습니다. Sam Altman은 미시간 Stargate 현장에 첫 철골 구조물이 세워졌다는 소식을 전했습니다. Oracle, Related Digital과 협력하는 이 대규모 AI 데이터센터가 이제 물리적 형태를 갖추기 시작한 건데요, OpenAI의 수십억 달러 인프라 투자가 본격화되고 있음을 보여줍니다. OpenAI Developers 계정은 gpt-realtime-1.5를 사용한 음성 에이전트 데모를 공유했습니다. 싱가포르 의료 클리닉 접수용으로, 환자와 자연스럽게 대화하며 예약을 실시간으로 처리하는 모습이 인상적이었습니다. 그리고 Felix Rieseberg는 Cowork에서 발생한 ECONNRESET 오류의 근본 원인을 48시간 추적 끝에 찾았다며, 결론은 역시나 DNS 문제였다고 밝혔습니다. 개발자라면 공감할 만한 이야기죠.

오늘의 GEEK DAILY를 정리하겠습니다. AI 코딩 에이전트 시장이 Claude Code, Codex, Z.AI의 삼파전으로 확대되면서 개발자 도구의 진화가 가속화되고 있고, 230년 된 기업 STADLER의 사례는 AI 도입이 더 이상 선택이 아닌 필수임을 보여주고 있습니다. 그리고 Windows 11의 Microsoft 계정 강제 정책도 변화의 조짐이 보이네요. 오늘의 GEEK DAILY 매거진은 여기까지입니다. 내일도 최신 테크 소식으로 찾아뵙겠습니다. 감사합니다.
